# Store Assets — чек-лист

Изображения не сгенерированы — здесь требования и сценарии для съёмки.
Статус исходников: `assets/icon.png` (512×512) есть; `assets/images/` пуст.

## Иконки

- [x] Исходник ≥128×128: `assets/icon.png` (512×512 PNG) — есть
- [x] 16/32/48/64/128 в манифесте — Plasmo генерирует автоматически из
  `assets/icon.png` (проверено в `build/chrome-mv3-prod/manifest.json`)
- [ ] **Store icon 128×128 PNG** для формы листинга — экспортировать из
  `assets/icon.png` (Store требует загрузить отдельно; фон не должен быть
  прозрачным по краям до обреза — проверить на светлой и тёмной теме)

## Скриншоты (минимум 1, лучше 3–5)

Формат: **1280×800** (предпочтительно) или 640×400, PNG/JPEG, без
прозрачности. Снимать на реальном MR с непустым списком обсуждений;
замазать/заменить внутренние URL и имена, если снимаем на приватном инстансе.

- [ ] **S1 — «Сайдбар на странице MR»** (главный кадр): открытый MR на
  gitlab.com, справа сайдбар со списком из 3–5 задач — видны автор, превью
  комментария, file:line, счётчик «N open · M resolved».
- [ ] **S2 — «Send to AI»**: крупный план карточки задачи с кнопкой в
  состоянии «✓ Copied»; рядом (вторым окном) — вставленный в AI-инструмент
  промпт с разделами `## Review comment` / `## Diff hunk`.
- [ ] **S3 — «Открытые vs resolved»**: тот же сайдбар с включённым
  тумблером «Show resolved» — resolved-задачи приглушены, бейдж RESOLVED.
- [ ] **S4 (опц.) — «Пустое состояние»**: MR без открытых задач — «No open
  review tasks. 🎉» (показывает аккуратность UI).
- [ ] **S5 (опц.) — «Тред с ответами»**: задача с «+N replies», чтобы было
  видно, что копируется весь тред, а не один комментарий.

## Promo tiles (опционально)

- [ ] Small promo tile **440×280** PNG/JPEG: иконка + слоган
  «MR comments → AI prompts» на однотонном фоне
- [ ] Marquee **1400×560** (нужен только для фичеринга — можно пропустить)

## Общие требования

- [ ] Ни на одном изображении нет чужих торговых марок крупным планом
  (логотип GitLab в интерфейсе страницы — ок, в иконке/промо — нет)
- [ ] Нет приватных данных (имена коллег, внутренние hostname'ы, код под NDA)
- [ ] Скриншоты соответствуют текущей версии UI (сверить с v0.2.2)
